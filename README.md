# CGCNN PyG + KAN

This repository contains a compact PyTorch Geometric CGCNN implementation where
the interaction network inside each CGCNN convolution can be switched between:

- `mlp`: a standard MLP that maps `[x_i, x_j, edge_attr]` to CGCNN gate/core
  features.
- `kan`: a smaller B-spline KAN block in the same position.

The graph-level readout is kept as a normal MLP in both variants, so the
benchmark compares the CGCNN-internal MLP replacement rather than replacing only
the final readout head.

## Install

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -U pip
.\.venv\Scripts\python -m pip install -r requirements.txt
```

## Benchmark Dataset

The current benchmark uses a deterministic synthetic crystal-like graph
regression dataset generated by [`data.py`](cgcnn_pyg_kan/data.py).
It is not Materials Project, JARVIS, QM9, or another real materials benchmark.

Each graph contains random atom features, random 3D positions, distance-cutoff
edges, Gaussian distance edge features, and a target built from nonlinear atom
terms plus an edge-distance term. This makes it useful for a quick sanity and
speed benchmark, but it is not a publishable materials-performance result.

## Run Benchmark

```powershell
.\.venv\Scripts\python scripts\benchmark.py --conv-nets mlp kan --epochs 10 --graphs 512
```

Each run writes JSON and CSV files into `benchmarks/`.

Latest local benchmark, run on CPU with `torch 2.12.1+cpu` and
`torch-geometric 2.8.0`:

```powershell
.\.venv\Scripts\python scripts\benchmark.py `
  --conv-nets mlp kan `
  --epochs 30 `
  --graphs 512 `
  --batch-size 32 `
  --conv-kan-hidden-dim 16 `
  --conv-kan-grid-size 3 `
  --forward-iters 80 `
  --warmup-iters 10
```

| conv net | params | best val MAE | test MAE | test RMSE | train seconds | forward ms/batch |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| mlp | 123,329 | 0.070210 | 0.076663 | 0.097308 | 52.924 | 33.283 |
| kan | 122,657 | 0.058214 | 0.071456 | 0.086130 | 333.995 | 285.235 |

Useful knobs for making KAN smaller:

```powershell
.\.venv\Scripts\python scripts\benchmark.py `
  --conv-nets mlp kan `
  --hidden-dim 64 `
  --head-hidden-dims 128 64 `
  --conv-kan-hidden-dim 16 `
  --conv-kan-grid-size 3 `
  --conv-kan-spline-order 3
```

## Model Usage

```python
from cgcnn_pyg_kan import CGCNN

model = CGCNN(
    node_input_dim=10,
    edge_input_dim=16,
    hidden_dim=64,
    num_convs=3,
    head_hidden_dims=(128, 64),
    conv_net="kan",          # or "mlp"
    conv_kan_hidden_dim=16,  # smaller KAN bottleneck
    conv_kan_grid_size=3,
)
```
